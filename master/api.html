API Reference¶

The following section outlines the API of the data downloaded from OSM

OSM Data¶

class osm_data.OSM_data(loggin_level: int = 20, skip_update: bool = False, first_node_has_trash: bool = False)¶

Represents the OSM Data in an easy to access format You can access the data with .ways (and if needed with ._nodes)

logging_level: int

Sets the logging level used by the logging module
Defaults to INFO

skip_update: bool

Defines if the update check should be skipped
Defaults to False

Warning

Please check for updates manually regularly if you disable autoupdate

first_node_has_trash: bool

Defines if starting node should always have a trash container Defaults to False

data_file: string

The Name of the datafile that should be used Defaults to data.lposm

logger¶

logger logging The logger used by the OSM module

New in version 0.2.3.

filtered_nodes(filter: str)¶

Returns a list of Node that are filtered based on the given Parameter

filter: str

Sets the current parameters that should be filtered out

Parameter Options

used - Return only nodes that have at least one way connected
trash - Return only nodes that have trash and at least one way connected

NotImplementedError

You gave an invalid filter option
Request filter options here: https://github.com/Abfall-Karte/Abfallkarte/issues

Node¶

class osm_data.Node(id: int, lat: float, long: float, trash: bool, attributes: tuple = {})¶

Represents the Node in the OSM data set

id: int

The id of the Node

lat: float

The Longitude location of the Node

lon: float

The Latitude Location of the Node

trash: bool

Defines the if the current node has a Trash container

attributes: tulpe

The attributes of the Node

id¶

id: int The unique id of this node

Warning

The id of the node is not unique. There will be several nodes with the same id but different attributes. Use subid for a unique id system

lon¶

long: float The Longitude location of the Node

lat¶

lat: float The Latitude Location of the Node

trash¶

trash: bool Defines if this Node has a Trash container

attributes¶

attributes: bool All attributes. Usually not important

parent¶

parent: osm_data For internal use only The parent that has all the ways. Used to reference the ways

subid¶

subid: int A second unique identifier which is set to None by Default. Requested

property type¶

type: NodeType Returns the type of the Node

The length of the way (unitless)

Defines the max speed on the way

name: string The streetname

id¶

id: int Defines the unique id of the Way

length¶

length: int The node where this street ends

x__nodes¶

nodes: list[Node] A list of all the nodes in the current street

speed¶

speed: int The maximum allowed speed on this street

property weight¶

weight: float The weight of the way

property start¶

start: Node The node where this street starts

property end¶

end: Node The node where this street ends

NodeType¶

class osm_data.NodeType(value)¶

Defines the type of the a Node

invalid = <NodeType.invalid: -1>¶

The node is not connected to any streets

street_end = <NodeType.street_end: 0>¶

The node is connected with one node

street = <NodeType.street: 1>¶

The node is connected with two nodes

junction = <NodeType.junction: 2>¶

The nodes is connected with three or more nodes
