API Reference¶

The following section outlines the API of the data downloaded from OSM

OSM Data¶

class osm_data.OSM_data(loggin_level: int = 20, skip_update: bool = False, first_node_has_trash: bool = False)¶

Represents the OSM Data in an easy to access format You can access the data with .ways (and if needed with ._nodes)

Parameters

  • logging_level (int) – Sets the logging level used by the logging module Defaults to INFO

  • skip_update (bool) –

    Defines if the update check should be skipped Defaults to False

    Warning

    Please check for updates manually regularly if you disable autoupdate

  • first_node_has_trash (bool) – Defines if starting node should always have a trash container Defaults to False

logger¶

logger logging The logger used by the OSM module

New in version 0.2.3.

ways¶

ways list[ways] A List of all ways in the current dataset. This includes ways not connected to the main dataset.

blacklisted_ways_ids¶

blacklisted_ways_ids list[int] A list of all ways that nodes won’t return when asked for connected ways

route_set¶

route_set bool Defines if visualize has a route to show

route¶

route list[Node] A list of all nodes that make a Route

route_trash¶

route_trash list[Node] A list of all Trash nodes on the Route

filtered_nodes(filter: str)¶

Returns a list of Node that are filtered based on the given Parameter

Parameters

filter (str) –

Sets the current parameters that should be filtered out

Parameter Options

”used” - Return only nodes that have at least one way connected
“trash” - Return only nodes that have trash and at least one way connected

Raises

NotImplementedError – You gave an invalid filter option
Request filter options here: https://github.com/Abfall-Karte/Abfallkarte/issues

Returns

Return type

List[Node]

visualize(route: list = [], route_trash: list = []) → None¶

Creates a visualisation of the current dataset and displays it

Parameters

  • rout (list[Node]) – The lists of nodes forming the way that should be displayed

  • route_trash (list[Node]) – A list of Nodes that contain trash and should be marked accordingly. Needs to contain at least one Node

property updated_dataset¶

updated_dataset bool Defines if the dataset was updated. Must be manually set to false

update_dataset()¶

Updates the OSM Dataset

Note

The update process will take some time depending on your machine (at least 15 Minutes) It is strongly encouraged to run this with PyPy instead of CPython to reduce runtime

class osm_data.Node(id: int, lat: float, long: float, trash: bool, attributes: tuple = {})¶

Represents the Node in the OSM data set

:param : :param id int: The id of the Node :param lat float: The Longitude location of the Node :param lon float: The Latitude Location of the Node :param trash bool: Defines the if the current node has a Trash container :param attributes tulpe: The attributes of the Node

id¶

id int The unique id of this node

Warning

The id of the node is not unique. There will be several nodes with the same id but different attributes. Use subid for a unique id system

NodeType¶

class osm_data.NodeType(value)¶

Defines the type of the a Node

invalid = <NodeType.invalid: -1>¶

The node is not connected to any streets

street_end = <NodeType.street_end: 0>¶

The node is connected with one node

street = <NodeType.street: 1>¶

The node is connected with two nodes

junction = <NodeType.junction: 2>¶

The nodes is connected with three or more nodes
