Programm Documentation¶

This is a command line application. You can call the main function with python client.py. In the following documentation this will be shortened to just client.py. Then a function in Controller is called, which handles the behavior. In the Network class everything network related is handled.

Main Class¶

client.main()¶

This is the main method. The command line options are defined here. You can call this programm with client.py. Without arguments it will print out the helptext, which says, which arguments are possible. The programm is splitted in several subcommands. The helptext will show you them. With client.py SUBCOMMAND --help the helptext for the subcommand is printed. For sure you need to replace SUBCOMMAND with the desired subcommand.

Example usage: client.py route will ask the server for a route and print it as output.

Controller¶

class controller.Controller(confName)¶

The main controller of the programm. Every command will call a function in this class.

config(args)¶

This method handles the command line UI config options. It is called by client.py config.

create_config_file()¶

This method is called when no configfile is found or a restoring to the defaults with client.py config -d or client.py config --defauls is requested.

debug(args)¶

This method handles the debug commands. It is called by the subcommand client.py debug.

route(args)¶

This method calls the route() function from Network. This method is called by client.py route.

Network¶

class network.Network(confName)¶

This class handles all connections with the server.

askRoute()¶

This method asks the server for a route and prints the gathered route as output. You can call this function by client.py route.

ping()¶

This method starts a connection to the server and prints the answer. It can be used for debug reasions. This function can be called by client.py debug -p oder client.py debug --ping.

test()¶

This is a test method, which can be used to test some code. It can be called with client.py debug -t or client.py debug --test.

FileHandler¶
