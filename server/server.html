Programm Documentation¶

This is a command line application. You can call the main function with python server.py. In the following documentation this will be shortened to just server.py. Then a function in Controller is called, which handles the behavior. In the Network class everything network related is handled.

Main Class¶

The main module of the Routenfinder-Server.

server.main()¶

main method of the network server.py Calls the listen method in client.py so the serveris ready for connection.

Network¶

This Module contains a class with related network tasks.

class network.Network¶

This class contains methods related to networking tasks.

listen()¶

This method binds the server port and listens for input from the client It sends the messages beetween server and client and en- or decodes them. Also it closes the server-client connection.

ping()¶

This method returns a massage that confirms a successful ping.

route()¶

This method starts the routing algorithm that returns the best possible route.

switcher(msg)¶

This is a switcher method with returns the right method to the listen method.
